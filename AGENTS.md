# VisionMonitor 开发指令

本仓库是一个用于求职展示和学习的 Qt + C++ + OpenCV 工业视觉质检项目。后续在本仓库内进行代码新增、修改、重构或排错时，必须遵守下面的规则。

## 必须遵守

- 每次新增或修改 C++、CMake、脚本等代码时，都要同步补充必要注释。
- 注释要解释“为什么这样做”，尤其是平台兼容、线程安全、资源释放、异常兜底、图像算法和模型推理逻辑。
- 修改既有逻辑时，如果原注释已经过时，必须同步更新或删除，避免注释和实现不一致。
- 跨线程传递 `cv::Mat`、摄像头/串口打开释放、OpenCV 后端选择、YOLO 输出解析、NMS 和坐标换算等位置必须优先保留清晰注释。

## 注释风格

- 优先使用简短中文注释，面向正在学习 Qt/C++/OpenCV 的维护者。
- 不要给简单赋值、普通控件创建、直接函数调用添加机械复述式注释。
- 当代码依赖 Windows 特性或本机环境时，注释里要说明原因和可迁移风险。
- 提交前建议运行 `tools/check_comment_policy.ps1`，用脚本先过滤明显缺少注释的代码改动。
- 如果本地尚未启用 Git hook，先运行 `tools/install_git_hooks.ps1`，让提交前自动检查注释规则。

## 当前已知工程背景

- Windows 上 OpenCV 的 MSMF(Media Foundation) 摄像头链路在当前机器上出现过“能打开但读不到帧”的问题。
- 项目里的摄像头采集逻辑应优先验证 DirectShow，并且必须用实际非空帧判断后端是否可用，不能只看 `isOpened()`。
- `tools/camera_probe.cpp` 是摄像头后端诊断工具，后续改动时要保持它能独立验证 DirectShow、MSMF 和默认后端。
